Reusable Learning Object

Production-Ready LAMPP Stack on Ubuntu Server

This RLO walks learners through building a single Ubuntu Server virtual machine with Apache, MariaDB, PHP, and Python. It is designed as a guided, modular RLO with copy-friendly commands, verification checkpoints, troubleshooting notes, and evidence prompts.

Estimated total time: 90 to 120 minutes Format: web pages Audience: graduate IS students / early-career developers
Start Module 0 View references

Learning outcomes

By the end of this RLO, learners will be able to:

  1. Install Ubuntu Server on a virtual machine.
  2. Complete baseline server setup and hardening.
  3. Install and configure Apache and MariaDB.
  4. Deploy a simple PHP website.
  5. Install Python for server-side scripting and automation.

Before you begin

Prerequisite knowledgeBasic networking, terminal navigation, and a general understanding of web applications.
Required toolsA host machine capable of running a VM, a hypervisor such as VirtualBox/VMware, the latest Ubuntu Server LTS ISO, and internet access.
What this RLO coversSingle-server deployment inside a VM, SSH, UFW, Apache, MariaDB, PHP, and Python automation scripts.
What this RLO does not coverLoad balancing, clustering, WAFs, SELinux/AppArmor tuning, CI/CD pipelines, and TLS certificate deployment.

Module map

10 to 15 minutes

Module 0: Introduction and Planning

Goal

Define the target build and gather the required files before you touch the VM.

Success check

You understand the target architecture, have the required files and tools ready, and have created an evidence folder.

Open module
20 to 30 minutes

Module 1: VM Creation and Ubuntu Server Installation

Goal

Create the virtual machine, install Ubuntu Server, and verify that the VM boots and has an IP address.

Success check

The VM boots without the installer ISO, you can log in with the non-root account, and the VM has an IP address with internet access.

Open module
15 to 20 minutes

Module 2: Baseline Configuration and Hardening

Goal

Update the server, make sure SSH is available, disable direct root SSH login, and enable the firewall.

Success check

Package upgrades are installed, SSH is active, direct root SSH login is disabled, and UFW is enabled with OpenSSH allowed.

Open module
15 to 20 minutes

Module 3: Install Apache and Configure a Virtual Host

Goal

Install Apache2, create a dedicated site directory, configure a virtual host, and publish a placeholder page.

Success check

Apache is active, the configuration test returns Syntax OK, and the placeholder page loads at http://127.0.0.1:8080/.

Open module
15 to 20 minutes

Module 4: Install and Secure MariaDB

Goal

Install MariaDB, run the secure installation routine, create the demo database, and provision a least-privilege application account.

Success check

MariaDB is active, the lampp_demo database exists, and the lampp_user account can read from app_info.

Open module
10 to 15 minutes

Module 5: Install PHP and Connect it to Apache

Goal

Install PHP, the Apache PHP module, and the MariaDB driver. Confirm PHP executes in the browser.

Success check

php -v reports a PHP version, phpinfo.php loads in the browser, and Apache remains active after the restart.

Open module
15 to 20 minutes

Module 6: Deploy the Demo PHP Site

Goal

Replace the placeholder page with a PHP application that reads from MariaDB and reports a successful deployment.

Success check

The homepage loads at http://127.0.0.1:8080/, the page reports Database connection succeeded, and the seeded row from app_info is visible.

Open module
10 to 15 minutes

Module 7 - Install Python and Run an Automation Script

Goal

Install Python packaging tools, create a virtual environment, and run a script that checks the local web application.

Success check

The virtual environment is created, the script returns an HTTP 200 response, and the response preview shows HTML from the deployed site.

Open module
10 minutes

Module 8: Final Validation and Evidence Package

Goal

Confirm that every layer of the stack works and gather a clean evidence package.

Success check

All required services are running, the site validates end to end, and the evidence package is organized for submission.

Open module

Success criteria

Use this final check to confirm the environment works end to end and your evidence package is complete.

  • The site is reachable from a browser at http://127.0.0.1:8080/.
  • The demo PHP site confirms a successful MariaDB connection.
  • UFW is enabled and only the required access is open.
  • Python automation can request the local site and return an HTTP 200 response with an HTML preview.
  • Your evidence package includes the screenshots captured across Modules 1 to 7 and is organized for submission.
