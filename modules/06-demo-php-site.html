Module 6: Deploy the Demo PHP Site

Replace the placeholder page with a PHP application that reads from MariaDB and reports a successful deployment.

Estimated time: 15–20 minutes
Module progress
0 of 0 checkpoints completed

Your checklist progress is stored in your browser only on this device.

What you will complete

  • Store database credentials outside the public document root.
  • Deploy a PHP page that connects to MariaDB using PDO.
  • Show a visible success message in the browser.

Verification checklist

  • The homepage loads at http://VM-IP/.
  • The page displays Database connection succeeded.
  • The seeded row from app_info is visible in the browser.

Steps

Create a private configuration file

Do not place database credentials in the public document root. Keep the configuration in a separate directory so the web server serves only the application entry point and not the raw credentials file.

1. Create the non-public configuration directory

  1. Create /var/www/lampp-demo/config.
    Create the private config directory
    mkdir -p /var/www/lampp-demo/config
  2. Keep this directory alongside public, not inside it.
  3. This structure helps separate private configuration from browser-accessible files.

2. Create the database configuration file

  1. Create /var/www/lampp-demo/config/db.php.
    Create the database config file
    cat <<'PHP' > /var/www/lampp-demo/config/db.php
<?php
return [
    'dsn' => 'mysql:host=localhost;dbname=lampp_demo;charset=utf8mb4',
    'username' => 'lampp_user',
    'password' => 'ChangeThisPassword!',
];
PHP
  2. Store the DSN, username, and password in the returned PHP array.
  3. Use the exact password you assigned to lampp_user in Module 4.

3. Double-check the values before testing

  1. Confirm the database name is lampp_demo.
  2. Confirm the username is lampp_user.
  3. Confirm the host is localhost so the application connects to the local MariaDB service.

Create the application page

The application page is where the stack comes together. It should load the private config file, connect to MariaDB with PDO, fetch the seeded row, and display a clear success message in the browser.

1. Build the main PHP file

  1. Create /var/www/lampp-demo/public/index.php using the template below.
    Create the demo application page
    cat <<'PHP' > /var/www/lampp-demo/public/index.php
<?php
$config = require __DIR__ . '/../config/db.php';

try {
    $pdo = new PDO($config['dsn'], $config['username'], $config['password'], [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]);

    $stmt = $pdo->query('SELECT app_name, deployed_at FROM app_info LIMIT 1');
    $row = $stmt->fetch();
    $status = 'Database connection succeeded';
} catch (Throwable $e) {
    http_response_code(500);
    $status = 'Database connection failed';
    $row = null;
    $error = $e->getMessage();
}
?>
<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>LAMPP Demo Site</title>
  <style>
    body { font-family: Arial, sans-serif; margin: 2rem; line-height: 1.5; background: #f8fafc; color: #0f172a; }
    .card { background: white; max-width: 44rem; padding: 2rem; border-radius: 16px; box-shadow: 0 10px 25px rgba(15,23,42,.08); }
    .ok { color: #166534; }
    .bad { color: #991b1b; }
    code { background: #e2e8f0; padding: .2rem .35rem; border-radius: 6px; }
  </style>
</head>
<body>
  <div class="card">
    <h1>LAMPP Demo Site</h1>
    <p>This page confirms that Apache, PHP, and MariaDB are working together on Ubuntu Server.</p>
    <p class="<?= isset($error) ? 'bad' : 'ok' ?>"><strong><?= htmlspecialchars($status, ENT_QUOTES, 'UTF-8') ?></strong></p>
    <?php if ($row): ?>
      <ul>
        <li><strong>Application:</strong> <?= htmlspecialchars($row['app_name'], ENT_QUOTES, 'UTF-8') ?></li>
        <li><strong>Seeded at:</strong> <?= htmlspecialchars($row['deployed_at'], ENT_QUOTES, 'UTF-8') ?></li>
      </ul>
    <?php else: ?>
      <p><strong>Error:</strong> <code><?= htmlspecialchars($error ?? 'Unknown error', ENT_QUOTES, 'UTF-8') ?></code></p>
    <?php endif; ?>
  </div>
</body>
</html>
PHP
  2. Load the credentials file using a relative path that points from public to ../config/db.php.
  3. Create a PDO connection using the values from the config array.

2. Query the seeded database row

  1. Run a simple query against app_info.
  2. Fetch the first row and store it in a variable the page can display.
  3. Set a visible success message such as Database connection succeeded when the query works.

3. Include basic error handling

  1. Wrap the connection and query logic in a try/catch block.
  2. If the connection fails, return an HTTP 500 response and display a clear failure message.
  3. This makes troubleshooting much easier than a blank page or raw PHP warning.

4. Make the PHP page the default homepage

  1. Remove the old index.html placeholder file so Apache serves index.php by default.
    Remove the old placeholder page so index.php becomes the default
    rm -f /var/www/lampp-demo/public/index.html
  2. Keep only the files needed for the live demo page inside the public document root.

Lock down the file permissions

Once the application files are in place, set permissions that let Apache read the content without making the project writable by everyone.

1. Apply ownership deliberately

  1. Set ownership so your administrative user can still maintain the project files.
    Set project ownership
    sudo chown -R $USER:www-data /var/www/lampp-demo
  2. Use the web server group so Apache can read the application and configuration files.

2. Apply safe permissions

  1. Use restrictive permissions on the project directories, such as 750, so only the owner and group can access them.
    Set restrictive project permissions
    sudo chmod -R 750 /var/www/lampp-demo
  2. Use a tighter permission such as 640 on db.php because it contains the database password.
    Protect the database config file
    sudo chmod 640 /var/www/lampp-demo/config/db.php
  3. Avoid 777 or any other world-writable setting.

3. Restart Apache and test the final page

  1. Restart Apache so it picks up the new homepage cleanly.
    Restart Apache after deploying the app
    sudo systemctl restart apache2.service
  2. Open http://VM-IP/ in a browser.
  3. Confirm the page shows the deployment message, the database success message, and the seeded row values from app_info.

Troubleshooting

IssueWhat to check
500 Internal Server ErrorCheck the Apache error log and confirm the database password in db.php matches the MariaDB user password.
Credentials file is not foundConfirm the relative path __DIR__ . '/../config/db.php' is correct and the file exists.
Access denied for MariaDB userReset the password or recreate the lampp_user account, then try again.

  • A screenshot or terminal capture showing the restrictive permissions applied to /var/www/lampp-demo/config/db.php.
  • A screenshot of the deployed homepage showing the application title and deployment message.
  • A screenshot of the page displaying Database connection succeeded.
  • A screenshot of the seeded row from app_info visible in the browser.
