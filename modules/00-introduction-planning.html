Module 0: Introduction and Planning

Define the target build and gather the required files before you touch the VM.

Estimated time: 10 to 15 minutes
Module progress
0 of 0 checkpoints completed

What you will complete

  • Identify the target stack and what each layer does.
  • Confirm tools, ISO, VM resources, and success criteria.
  • Prepare an evidence package plan for submission.

Verification checklist

  • You can explain the target architecture in one or two sentences.
  • You have downloaded the Ubuntu Server ISO and installed a hypervisor.
  • You created an evidence folder on your host machine.

Steps

Understand the target architecture

In this RLO, you will build an Ubuntu Server VM that hosts:
  • Apache as the web server
  • MariaDB as the relational database
  • PHP for dynamic web pages
  • Python for server-side automation
The goal is a production-oriented baseline: updated packages, non-root administration, SSH access, a firewall, one virtual host, one demo database, one PHP application, and one Python automation script.

Gather the required resources

Before you build the VM, prepare the two host-side items you need: the Ubuntu Server LTS ISO and a working hypervisor. The steps below use VirtualBox on Windows, which is a common RLO setup.

Focus on the page structure, not the exact version number: download pages change over time, so use the screenshots as layout references even if the release shown in the example is older than the current one.

1. Download the Ubuntu Server ISO

  1. Open the official Ubuntu Server download page in your host browser.
  2. Choose the most recent Ubuntu Server LTS release, not the desktop edition.
  3. Download the 64-bit AMD64 ISO file.
  1. Save the file somewhere easy to find later, such as Downloads or a dedicated isos folder.
  2. Wait for the download to finish completely before moving on.
  1. Confirm that the downloaded file ends in .iso so you know you have the installer image and not a web shortcut.

2. Install VirtualBox on the host machine

  1. Open the official VirtualBox downloads page in your browser.
  2. Under VirtualBox Platform Packages, select Windows hosts to download the Windows installer.
  1. Wait for the installer file to download completely before opening it.
  1. Open File Explorer and go to the folder where the installer was saved, such as Downloads.
  2. Double-click the VirtualBox installer to start setup. If Windows shows a security or User Account Control prompt, allow it to continue.
  1. When the Oracle VirtualBox installer opens, review the welcome screen and click Next.
  1. Review the license agreement, choose the option to accept it, and click Next.
  1. Review the Custom Setup screen, confirm the install location and features you want, and then click Next.
  1. If the installer warns that your network connection will be interrupted briefly, make sure nothing important is using the network and click Yes to continue.
  1. Review the startup options, such as adding a Start menu entry or desktop shortcut, and click Next.
  1. At the Ready to Install screen, click Install.
  1. Wait while the installer copies files and configures VirtualBox. Approve any driver prompts if Windows asks for confirmation.
  1. When the installation completes, leave Start Oracle VirtualBox checked if you want it to open immediately, then click Finish.
  1. Confirm that Oracle VirtualBox Manager opens without errors and shows the main toolbar with options such as New. That means the host is ready for the VM creation steps in Module 1.

3. Confirm that the host is ready for Module 1

  1. Open File Explorer and make sure you can locate the Ubuntu ISO on the host machine.
  1. Open VirtualBox and confirm that the main manager window loads without errors.
  1. Click New once to confirm the VM creation wizard opens.
  1. Keep at least 25 GB of free disk space available on the host for the virtual disk and snapshots.

For this guided RLO, a practical VM baseline is:

  • 2 vCPUs
  • 4 GB RAM
  • 25 GB dynamically allocated disk
  • NAT networking for internet access

Create an evidence checklist before starting

Keep a folder on your host machine for screenshots and notes. Capture evidence as you go instead of trying to recreate it later. A good evidence package for this RLO includes:
  • VM settings screen
  • Ubuntu login prompt and IP address
  • Firewall status
  • Apache page loading in a browser
  • MariaDB login and database listing
  • PHP demo page showing a successful database connection
  • Python script output

Evidence to capture

  • A note stating the target architecture and what each component will do in the VM.
  • A screenshot showing the downloaded Ubuntu Server ISO on the host machine.
  • A screenshot showing the installed hypervisor open and ready to create a new VM.
  • A note or screenshot capturing the planned VM baseline.
  • A screenshot of the evidence folder you will use to store screenshots and notes for the rest of the RLO.
