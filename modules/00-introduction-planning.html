Hands-on module

Module 0 — Introduction and Planning

Define the target build, gather the required files, and decide what “done” looks like before you touch the VM.

Estimated time: 10–15 minutes Verification-driven Copy-friendly commands

What you will complete

  • Identify the target stack and what each layer does.
  • Confirm tools, ISO, VM resources, and success criteria.
  • Prepare an evidence package plan for submission.

Verification checklist

  • You can explain the target architecture in one or two sentences.
  • You have downloaded the Ubuntu Server ISO and installed a hypervisor.
  • You created an evidence folder on your host machine.

Steps

Understand the target architecture

In this RLO, you will build a **single Ubuntu Server VM** that hosts:
  • Apache as the web server
  • MariaDB as the relational database
  • PHP for dynamic web pages
  • Python for light server-side automation
The goal is a production-oriented baseline: updated packages, non-root administration, SSH access, a firewall, one virtual host, one demo database, one PHP application, and one small Python automation script.

Gather the required resources

Download the latest Ubuntu Server LTS ISO available when you complete the lab. Install your virtualization platform first (for example, VirtualBox or VMware). For this guided lab, a practical baseline is:
  • 2 vCPUs
  • 4 GB RAM
  • 25 GB dynamically allocated disk
  • NAT networking for internet access
These values are tutorial recommendations for a smooth lab experience rather than hard minimums.

Create an evidence checklist before starting

Keep a folder on your host machine for screenshots and notes. Capture evidence as you go instead of trying to recreate it later. A good evidence package for this RLO includes:
  • VM settings screen
  • Ubuntu login prompt and IP address
  • Firewall status
  • Apache page loading in a browser
  • MariaDB login and database listing
  • PHP demo page showing a successful database connection
  • Python script output

Command guide

Check the VM IP address after install
hostname -I
Confirm the OS release
cat /etc/os-release

Troubleshooting

IssueWhat to check
Not enough disk space on the hostMove the VM storage location to a drive with more free space before installation.
No internet in the VM later onKeep NAT enabled for the installation phase. You can experiment with bridged networking after the base stack works.

Evidence to capture

Capture a screenshot of your VM configuration and a note listing the stack components you plan to install.
