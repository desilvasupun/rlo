Module 7 — Install Python and Run a Small Automation Script

Install Python packaging tools, create a virtual environment, and run a script that checks the local web application.

Estimated time: 10–15 minutes Verification-driven Copy-friendly commands

What you will complete

  • Install Python package-management support.
  • Create and activate a virtual environment.
  • Run a small health-check automation script.

Verification checklist

  • The virtual environment is created successfully.
  • The script prints an HTTP 200 status line.
  • The response preview shows HTML from the deployed site.

Steps

Install Python support packages

Ubuntu already includes Python, but for a repeatable workflow you should also install python3-pip and python3-venv.

Create a virtual environment

Put the Python automation project under /opt/lampp-demo-python and create a dedicated virtual environment there. This keeps the automation files separate from the web application itself.

Create and run the health check

The script below uses Python’s built-in urllib package to request the local homepage and print the HTTP status code with a timestamp. No third-party modules are required.

Command guide

Install Python support packages
sudo apt install python3-pip python3-venv -y
Create a project directory
sudo mkdir -p /opt/lampp-demo-python && sudo chown -R $USER:$USER /opt/lampp-demo-python
Create a virtual environment
cd /opt/lampp-demo-python && python3 -m venv .venv
Activate the virtual environment
cd /opt/lampp-demo-python && source .venv/bin/activate
Create the health-check script
cat <<'PY' > /opt/lampp-demo-python/check_site.py
from datetime import datetime
from urllib.request import urlopen

url = 'http://127.0.0.1/'
now = datetime.utcnow().isoformat(timespec='seconds') + 'Z'

with urlopen(url, timeout=5) as response:
    body = response.read(200).decode('utf-8', errors='replace')
    print(f'[{now}] HTTP {response.status} from {url}')
    print('Preview:')
    print(body[:120])
PY
Run the health-check script
cd /opt/lampp-demo-python && source .venv/bin/activate && python check_site.py
Deactivate the virtual environment when finished
deactivate

Troubleshooting

IssueWhat to check
python3 -m venv failsMake sure python3-venv is installed.
HTTP request times outConfirm Apache is active and the site loads locally in the browser or with curl http://127.0.0.1/.
Permission denied under /optUse sudo chown -R $USER:$USER /opt/lampp-demo-python before creating files there.
