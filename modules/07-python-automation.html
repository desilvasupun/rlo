Module 7 - Install Python and Run an Automation Script

Install Python packaging tools, create a virtual environment, and run a script that checks the local web application.

Estimated time: 10 to 15 minutes
Module progress
0 of 0 checkpoints completed

What you will complete

  • Install Python package-management support.
  • Create and activate a virtual environment.
  • Run a small health-check automation script.

Verification checklist

  • The virtual environment is created successfully.
  • The script prints an HTTP 200 status line.
  • The response preview shows HTML from the deployed site.

Background

Python virtual environments isolate project-specific tooling from the system interpreter, which makes automation work easier to reproduce without changing the server's global Python environment. That isolation is useful for lightweight health checks like the one in this module because the script can be rerun later with the same local setup [14].

Steps

Install Python support packages

Ubuntu includes Python itself, but this module also needs package-management and virtual-environment support so the automation workflow is isolated and repeatable.

1. Install the Python tooling packages

  1. Run sudo apt install python3-pip python3-venv -y.
    Install Python support packages
    sudo apt install python3-pip python3-venv -y
  2. Wait for the package installation to finish fully.
  3. Confirm there are no package errors before creating the project directory.

2. Understand the purpose of the tools

  1. python3-pip gives you Python package installation support if you need it later.
  2. python3-venv allows you to create isolated Python environments for small automation projects.
  3. Even though this demo does not install third-party modules, using a virtual environment is still good practice.

Create a virtual environment

Keep the automation project separate from the web application. The Python script belongs under /opt/lampp-demo-python, and the virtual environment should live inside that project folder.

1. Create the project directory

  1. Create /opt/lampp-demo-python.
    Create the project directory
    sudo mkdir -p /opt/lampp-demo-python
  2. Change ownership to your normal user so you can create files there without running the project as root.
    Give your user ownership of the project folder
    sudo chown -R $USER:$USER /opt/lampp-demo-python
  3. Verify that you can enter the directory and list its contents.
    Enter the project directory
    cd /opt/lampp-demo-python
    List the project directory contents
    ls

2. Create the virtual environment

  1. From inside the project directory, run python3 -m venv .venv.
    Create a virtual environment
    cd /opt/lampp-demo-python && python3 -m venv .venv
  2. Wait for the environment creation to finish. This creates an isolated Python interpreter and supporting scripts.
  3. Confirm the .venv directory now exists.
    Show hidden files in the project directory
    ls -a

3. Activate it before running Python commands

  1. Run source .venv/bin/activate from the project directory.
    Activate the virtual environment
    cd /opt/lampp-demo-python && source .venv/bin/activate
  2. Notice that the shell prompt changes to show the active environment.
  3. Keep that environment active while creating and running the health-check script.

Create and run the health check

The goal of the script is simple: request the locally hosted site and report whether the stack returns a successful HTTP response. Using the built-in urllib module keeps the example lightweight.

1. Create the script file

  1. Create /opt/lampp-demo-python/check_site.py.
    Create the health-check script
    cat <<'PY' > /opt/lampp-demo-python/check_site.py
from datetime import datetime
from urllib.request import urlopen

url = 'http://127.0.0.1/'
now = datetime.utcnow().isoformat(timespec='seconds') + 'Z'

with urlopen(url, timeout=5) as response:
    body = response.read(200).decode('utf-8', errors='replace')
    print(f'[{now}] HTTP {response.status} from {url}')
    print('Preview:')
    print(body[:120])
PY
  2. Use Python code that sends a request to http://127.0.0.1/.
  3. Include a timestamp in the output so the result looks like a real automation check.
  4. Print both the HTTP status code and a short preview of the returned HTML.

2. Run the script inside the virtual environment

  1. With the virtual environment still active, run python check_site.py.
    Run the health-check script
    cd /opt/lampp-demo-python && source .venv/bin/activate && python check_site.py
  2. Confirm the script prints an HTTP 200 response.
  3. Confirm the response preview contains HTML from the deployed site rather than an error page.

3. Close the environment cleanly

  1. Run deactivate after the test completes.
    Deactivate the virtual environment when finished
    deactivate
  2. This returns the shell to the system Python environment.
  3. Keep the script and environment in place so you can rerun the health check during final validation.

Troubleshooting

IssueWhat to check
python3 -m venv failsMake sure python3-venv is installed.
HTTP request times outConfirm Apache is active and the site loads locally in the browser or with curl http://127.0.0.1/.
Permission denied under /optUse sudo chown -R $USER:$USER /opt/lampp-demo-python before creating files there.
