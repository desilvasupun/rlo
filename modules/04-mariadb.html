Module 4: Install and Secure MariaDB

Install MariaDB, run the secure installation routine, create the demo database, and provision a least-privilege application account.

Estimated time: 15 to 20 minutes
Module progress
0 of 0 checkpoints completed

What you will complete

  • Install MariaDB server and client tools.
  • Run the secure installation step.
  • Create a database, table, and application user.

Verification checklist

  • MariaDB is active according to systemd.
  • The lampp_demo database exists.
  • The lampp_user account can read from app_info.

Background

MariaDB should be secured before an application account is created or used. The secure-install utility removes common insecure defaults, and the SQL account-management statements used later in the module support a least-privilege design in which the application receives only the access it needs [7], [8], [9], [10].

Steps

Install MariaDB packages

MariaDB provides the relational database layer for the demo application. Install both the server and client tools so you can manage the database locally from the Ubuntu shell.

1. Install the required packages

  1. Run sudo apt install mariadb-server mariadb-client -y.
    Install MariaDB packages
    sudo apt install mariadb-server mariadb-client -y
  2. Wait for the package install to complete without errors.
  3. Make sure the command finishes before moving to any SQL work.

2. Start and enable the service

  1. Run sudo systemctl enable --now mariadb so the database starts now and also after future reboots.
    Enable and start MariaDB
    sudo systemctl enable --now mariadb
  2. Check the service status to confirm MariaDB is active.
    Check the MariaDB service status
    sudo systemctl status mariadb --no-pager
  3. If the service fails to start, troubleshoot that first rather than attempting database creation.

Run the secure installation step

Before creating application data, apply MariaDB's built-in baseline hardening. The goal is to remove common insecure defaults on a fresh install.

1. Launch the helper

  1. Run sudo mariadb-secure-installation.
    Run the secure installation helper
    sudo mariadb-secure-installation
  2. Read each prompt carefully because the script is interactive.

2. Work through the common security prompts

  1. When the helper asks for the current MariaDB root password on this fresh install, press Enter because you have not set one yet.
  2. When it asks whether to switch to unix_socket authentication, type n to not switch.
  3. When it asks whether to change the root password, type Y, then enter and confirm the new MariaDB root password.
  4. When it asks whether to remove anonymous users, type Y. This removes accounts that could otherwise log in without a named user.
  5. When it asks whether to disallow remote root login, type Y. This keeps the MariaDB root account limited to local administration instead of network access.
  6. When it asks whether to remove the test database and access to it, type Y. The test database is only for experimentation and should not remain on a server you are hardening.
  7. When it asks whether to reload the privilege tables now, type Y. This makes the security changes take effect immediately instead of waiting for a later reload or restart.

3. Understand the Ubuntu authentication model

  1. On Ubuntu, it is normal for the helper to say the root account is already protected before you answer the later password prompts. This is because the local MariaDB root account is often protected through Ubuntu's default socket-based administration model rather than relying only on a password.
  2. For the next section in this RLO, continue to use sudo mariadb for local administration from the server console.
  3. The helper removes insecure defaults.

Create the demo database and application user

Now create the sample data that the PHP application will read later. Keep the design simple: one database, one table, one seed row, and one application user with limited privileges.

1. Create the demo database objects

  1. Use the batch setup command below to create the demo objects in one pass.
    Create the database, table, seed row, and least-privilege user
    sudo mariadb <<'SQL'
CREATE DATABASE lampp_demo;
USE lampp_demo;
CREATE TABLE app_info (
    id INT AUTO_INCREMENT PRIMARY KEY,
    app_name VARCHAR(100) NOT NULL,
    deployed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
INSERT INTO app_info (app_name) VALUES ('LAMPP Demo Site');
CREATE USER 'lampp_user'@'localhost' IDENTIFIED BY 'ChangeThisPassword!';
GRANT SELECT ON lampp_demo.* TO 'lampp_user'@'localhost';
FLUSH PRIVILEGES;
SQL
  2. Create a database named lampp_demo.
  3. Switch into that database with USE lampp_demo;.
  4. Create the app_info table with an ID column, application name, and timestamp.
  5. Insert one seed row so the application has something to display.

2. Create the application login

  1. Create a user named lampp_user that connects from localhost.
  2. Assign a strong password and record it because you will need the same value in the PHP configuration later.
  3. Grant only SELECT access on lampp_demo so the account has the minimum rights needed for the demo site.
  4. Run FLUSH PRIVILEGES; after the grant statements.

3. Test the least-privilege account

  1. Connect using lampp_user and query app_info.
    Test the new application account
    mariadb -u lampp_user -p lampp_demo -e "SELECT * FROM app_info;"
  2. Confirm the row you inserted is returned successfully. That proves both the database content and the application account are ready for Module 6.

Troubleshooting

IssueWhat to check
Cannot log in as root with a passwordOn Ubuntu, try sudo mariadb. Socket authentication is common for the administrative account.
SQL syntax errorRun one statement at a time inside the MariaDB shell to isolate the failing line.
Application user cannot connectVerify the username, password, host (localhost), and that FLUSH PRIVILEGES; completed.
