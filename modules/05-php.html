Module 5: Install PHP and Connect It to Apache

Install PHP, the Apache PHP module, and the MariaDB driver, then confirm PHP executes in the browser.

Estimated time: 10–15 minutes
Module progress
0 of 0 checkpoints completed

Your checklist progress is stored in your browser only on this device.

What you will complete

  • Install PHP and the required packages.
  • Restart Apache so PHP execution is active.
  • Confirm that PHP runs correctly.

Verification checklist

  • The php -v command reports a PHP version.
  • Visiting http://VM-IP/phpinfo.php loads a PHP info page before you remove it.
  • Apache remains active after restart.

Steps

Install PHP packages

Install PHP itself, the Apache integration module, the CLI package, and the MariaDB driver package.

Restart Apache and test PHP

Create a temporary phpinfo.php file in the site document root to verify that Apache is executing PHP. Remove it afterward because it exposes detailed environment information.

Command guide

Install PHP and common packages for this RLO
sudo apt install php libapache2-mod-php php-cli php-mysql -y
Restart Apache after PHP installation
sudo systemctl restart apache2.service
Check the PHP version in the shell
php -v
Create a temporary PHP test file
cat <<'PHP' > /var/www/lampp-demo/public/phpinfo.php
<?php
phpinfo();
PHP
Remove the PHP info page after validation
rm /var/www/lampp-demo/public/phpinfo.php

Troubleshooting

IssueWhat to check
Browser downloads the PHP file instead of executing itConfirm libapache2-mod-php is installed and restart Apache.
Blank page in browserCheck /var/log/apache2/lampp-demo-error.log and the main Apache error log.
PHP command not foundThe package install may have failed. Rerun the package command and confirm network access.

Evidence to capture

Capture the PHP version in the terminal and one screenshot of the PHP info page before you delete it.
