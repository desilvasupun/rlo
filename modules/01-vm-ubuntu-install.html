Module 1 — VM Creation and Ubuntu Server Installation

Create the virtual machine, install Ubuntu Server, and verify that the VM boots and has an IP address.

Estimated time: 20–30 minutes Verification-driven Copy-friendly commands

What you will complete

  • Create a working Ubuntu Server VM.
  • Complete the installer with a normal admin account.
  • Confirm the VM boots and is network reachable.

Verification checklist

  • The VM boots into Ubuntu Server without the installer ISO.
  • You can log in with the non-root account you created.
  • The VM reports an IP address and can reach the internet.

Steps

Create the virtual machine

In your hypervisor, create a new 64-bit Ubuntu VM. Attach the Ubuntu Server ISO to the virtual optical drive. Use the resource profile from Module 0 unless your environment needs something else.
Tip: Name the VM something clear like ubuntu-lampp-lab so your screenshots and notes stay organized.

Complete the Ubuntu installer

Boot from the ISO and work through the installer. A straightforward lab-friendly setup is:
  1. Choose your language and keyboard layout.
  2. Use the standard Ubuntu Server install path.
  3. Accept DHCP networking if your NAT adapter provides it.
  4. Use guided storage unless your course requires custom partitioning.
  5. Create a normal administrative user account.
  6. Select OpenSSH server during installation if the option is presented.
When installation finishes, reboot and detach the ISO if the VM keeps returning to the installer.

Log in and collect the first validation data

Log in with the account you created during installation and record the machine IP address. If you intend to SSH in from the host later, note the address now.

Command guide

Display IPv4/IPv6 addresses
hostname -I
Show network interface details
ip a
Confirm that the VM can reach the internet
ping -c 4 ubuntu.com

Troubleshooting

IssueWhat to check
VM loops back to installerPower off the VM and remove the ISO from the virtual optical drive.
No IP address shownCheck that the virtual NIC is attached and enabled. In NAT mode, a reboot often renews the DHCP lease.
Ping failsConfirm your host has internet access and the VM NIC is connected. NAT is the easiest mode for the initial build.

Evidence to capture

Capture the Ubuntu login prompt and a terminal showing the IP address from hostname -I.
