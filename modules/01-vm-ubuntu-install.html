Module 1: VM Creation and Ubuntu Server Installation

Create the virtual machine, install Ubuntu Server, and verify that the VM boots and has an IP address.

Estimated time: 20 to 30 minutes
Module progress
0 of 0 checkpoints completed

What you will complete

  • Create a working Ubuntu Server VM.
  • Complete the installer with a normal admin account.
  • Confirm the VM boots and is network reachable.

Verification checklist

  • The VM boots into Ubuntu Server without the installer ISO.
  • You can log in with the non-root account you created.
  • The VM reports an IP address and can reach the internet.

Background

Ubuntu Server provides the operating-system baseline for the rest of the stack, so a clean installation and a successful first boot matter before any hardening or application work begins. Ubuntu's installation guidance also treats the initial account, networking, and optional OpenSSH selection as foundational setup choices because they determine how the server will be administered after deployment [1].

Steps

Create the virtual machine

Build the VM carefully before you ever boot the installer. A clean VM definition prevents avoidable problems later, such as disk shortages, boot loops, or missing network access.

1. Start the VM wizard

  1. Open your hypervisor and choose the option to create a new virtual machine.
  2. Enter a clear name such as ubuntu-lampp-rlo so the VM is easy to identify in screenshots and later maintenance work.
  3. Select the downloaded Ubuntu Server ISO as the startup or installation media when the wizard asks for it.
  4. Make sure the operating system type is set to Linux and the version matches a 64-bit Ubuntu release.

2. Assign the baseline hardware

  1. Set the VM to use 2 vCPUs. That is enough for this RLO without placing unnecessary load on the host machine.
  2. Allocate 4 GB of RAM. This is a practical balance for Ubuntu Server, Apache, MariaDB, PHP, and Python tooling.
  3. Create a 25 GB virtual hard disk.
  4. Choose a dynamically allocated disk if that option is available so the file grows as needed instead of consuming the full size immediately.

3. Confirm the network and boot settings

  1. After the VM is created, review the summary page once so you can confirm the VM name, CPU count, RAM amount, disk size, and attached ISO.
  2. Open the VM settings before first boot and confirm the network adapter is enabled.
  3. Use NAT networking. NAT is the simplest choice for package downloads and outbound internet access from the guest.
  4. If you are using VirtualBox, open the NAT adapter's Port Forwarding settings before first boot.
  5. Add at least one HTTP forwarding rule so the host browser can reach Apache later: Host Port 8080 to Guest Port 80. If you want host-side SSH access later, also add Host Port 2222 to Guest Port 22.
  6. Check that the Ubuntu ISO is attached to the virtual optical drive, then save the settings and do one final review.
Important: With VirtualBox NAT, host-side browser tests in this RLO will use http://127.0.0.1:8080/.
Tip: Take a screenshot of the VM summary or settings page now. It is useful evidence for the build and helps if you need to recreate the machine later.

Complete the Ubuntu installer

Boot the VM from the ISO and work through the server installer carefully. The exact screen wording can vary slightly by Ubuntu release, but the overall flow stays consistent.

1. Work through the initial installer screens

  1. Start the VM and wait for the GRUB boot menu to appear. Choose Try or Install Ubuntu Server if the menu asks what you want to do.
  2. After the boot selection, Ubuntu may show a short sequence of startup messages while the installer loads. Wait for the installer interface instead of interrupting the boot process.
  3. Choose your preferred language.
  4. Select the correct keyboard layout and test it if the installer provides a keyboard check screen.

2. Accept the standard server installation path

  1. Choose the normal Ubuntu Server installation workflow rather than a minimized or customized path for this RLO.
  2. When the installer configures networking, accept the automatically assigned DHCP settings if the VM already shows a valid network connection.
  3. If the installer asks about a proxy and you do not use one in your environment, leave it blank.
  4. If the installer asks about the package mirror, keep the default mirror unless your environment provides a local mirror.

3. Configure storage and create the admin account

  1. Use the installer's guided storage option.
  2. Review the storage summary before confirming so you know Ubuntu will install onto the correct virtual disk.
  3. Create a normal user account with a username and password that you can remember for the rest of the RLO. This profile becomes your day-to-day administrative account on the server.
  4. If Ubuntu Pro is offered, choose Skip for now unless your environment specifically requires it.

4. Finish the install cleanly

  1. Select OpenSSH server during installation if the installer offers that checkbox. It saves time in the next module.
  2. Skip optional server snaps unless you specifically want them. They are not required for this RLO.
  3. Wait for package installation to finish fully. Do not reset or power off the VM during this stage.
  4. When the installer says the process is complete, choose Reboot Now.
  5. If the installer asks you to remove the installation medium, detach the ISO from settings if needed and press Enter.
  6. After reboot, confirm that the VM starts from the virtual disk and reaches the Ubuntu login prompt.

Log in and collect the first validation data

After the reboot, confirm that the server is usable before you continue to hardening and package installation.

1. Verify that Ubuntu boots normally

  1. Wait for the text login prompt to appear instead of the installer menu.
  2. Log in with the non-root account you created during installation.
  3. Confirm you can reach a normal shell prompt without errors.

2. Record the network information

  1. Run hostname -I to display the current IP address.
    Display IPv4/IPv6 addresses
    hostname -I
  2. Run ip a if you want to see which interface received the address.
    Show network interface details
    ip a

3. Confirm outbound connectivity

  1. Run ping -c 4 ubuntu.com to verify that the VM can reach the internet.
    Confirm that the VM can reach the internet
    ping -c 4 ubuntu.com
  2. If the ping succeeds, the machine is ready for package installation in the next module.
  3. If it fails, stop and fix networking now rather than building the rest of the stack on a disconnected VM.

Troubleshooting

IssueWhat to check
VM loops back to installerPower off the VM and remove the ISO from the virtual optical drive.
No IP address shownCheck that the virtual NIC is attached and enabled. In NAT mode, a reboot often renews the DHCP lease.
Ping failsConfirm your host has internet access and the VM NIC is connected. NAT is the easiest mode for the initial build.

Evidence to capture

  • A screenshot of the VM summary and NAT settings page showing the VM name, CPU count, RAM, disk size, NAT networking, and the port forwarding rules.
  • A screenshot showing the Ubuntu Server installation completed and the VM booting from the virtual disk instead of the installer ISO.
  • A screenshot of the Ubuntu login prompt and a successful login using the non-root administrative account you created.
  • A screenshot of a terminal showing the IP address from hostname -I.
  • A screenshot or note showing successful outbound connectivity from ping -c 4 ubuntu.com.
